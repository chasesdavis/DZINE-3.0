# Community Taste Packs

D-ZINE taste packs are meant to be portable, inspectable design judgment. A pack may describe a product style, industry posture, platform behavior, brand identity mode, or critique rubric.

## Package Shape

Community packs should be publishable as packages such as:

```text
@dzine/taste-linear-operator
@dzine/taste-brutalist-editorial
@dzine/taste-fintech-premium
@dzine/taste-mobile-health
```

Each pack must include:

- provenance and source URLs
- license notes
- allowed use
- extracted principles
- positive examples
- anti-patterns
- component rules
- typography rules
- layout rhythm rules
- copy rules
- screenshot/reference notes
- evaluation rubric

## Review Policy

Community taste packs are untrusted until reviewed. A reviewed pack should prove:

1. It does not copy protected assets, layouts, copy, or identity systems.
2. It cites sources and describes allowed use.
3. It converts inspiration into principles and contracts.
4. It includes positive and negative examples.
5. It can be audited against generated work.

## Local Learning Loop

D-ZINE may store local learnings after a user approves a critique. Local learnings should not automatically modify global profiles. Strong local learnings can be proposed as community pack updates after review.
