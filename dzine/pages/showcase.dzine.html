Screenshots

Responsive output is part of the proof, not an afterthought.

The visual audit captures mobile, tablet, and desktop screenshots. The proof can be reviewed by humans and compared over time.

mobile

tablet

Generated artifacts

The repo contains proof files for prompts, exports, screenshots, and audits.

D-ZINE is meant to make design quality reviewable in normal engineering workflows.

Prompt

`reports/website/site-prompt.txt` contains the compiled agent-ready prompt.

Exports

`reports/website/exports` contains tokens, CSS variables, Tailwind config, and compatibility Markdown.

Site audit

`reports/website/site-audit` contains route-level audit reports and screenshots.

Anti-slop proof

Evidence must be concrete enough to challenge the design.

Fake proof is worse than no proof. D-ZINE should show current artifacts and avoid invented numbers.

Rejected

Invented traction, fake reliability numbers, and screenshots that are not generated from the current repo.

Preferred

Current commands, current screenshots, generated reports, and clear notes about what has been verified.

Quality trail

The showcase is complete when the route audit passes.

Use `dzine site audit` to rebuild the site and inspect every route. The command writes a top-level site report and route-specific audit folders.

Command

Regenerate the proof.

node dist/cli/index.js site audit --config dzine.config.json --out reports/website/site-audit --port 6200
